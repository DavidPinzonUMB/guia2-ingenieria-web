Reflexiones sobre el proceso de diseño y desarrollo web.

1. La Importancia de la Especificidad en CSS

La especificidad en CSS es fundamental porque determina qué regla de estilo "gana" cuando hay múltiples selectores intentando modificar el mismo elemento. Entender cómo se calcula (etiquetas valen poco, las clases valen más y los IDs aún más) nos ayuda a escribir un código mantenible. En esta práctica, no se nos permitió usar ni clases ni IDs, lo que me obligó a aprovechar las combinaciones de selectores descendientes, adyacentes y de pseudo-clase. Esto demuestra que si la estructura HTML está bien planteada, puedes dar estilo a todo un proyecto sin depender exageradamente de clases (como a menudo pasa con frameworks tipo Tailwind), manteniendo tu CSS enfocado y sin peso muerto de selectores excesivos.

2. Selectores Avanzados Desafiantes

  • Selectores de hermanos adyacentes (+) y generales (~): Me costaron un poco entender porque dependen completamente de dónde esté ubicado el elemento a nivel estructural de "hermanos". Muchas veces quieres seleccionar un nodo que está dentro de otro contenedor y el botón general no llega hasta ahí, lo que obliga a replantear el DOM.
  • Pseudo-elementos (::before y ::after): Entender que se comportan como elementos que nacen "adentro" o "afuera" del texto, pero creados directamente desde el CSS sin existir en el HTML requirió ajustar mi lógica mental. Fue desafiante utilizarlos para poner decoradores antes de la lista, pero al entender que requerían atributos como "content" para aparecer, se hizo más sencillo.

3. El Paradigma Mobile-First

El concepto de Mobile-First no se trata solo de escribir el código CSS para un móvil y luego usar @media (min-width), sino de poner la lógica y simplicidad de la experiencia por delante. Los beneficios de esta mentalidad son enormes: nos fuerza, como desarrolladores, a centrarnos solo en el contenido crítico y en la funcionalidad principal que requiere el usuario (porque el espacio de la pequeña pantalla es muy limitado). Además, mejora el rendimiento ya que el dispositivo móvil no tiene que procesar hojas de estilo sumamente pesadas preparadas para un monitor de 4k y luego "deshacerlas".

Aplicación en el Perfil: En nuestro proyecto lo apliqué escribiendo por defecto un comportamiento display: flex con dirección en column. Esto acomoda inmediatamente en apilamiento vertical el encabezado (foto y nombre) y los datos de la cuenta. Posteriormente, coloqué una regla de media query de 768px que cambia el layout a CSS Grid de dos columnas, logrando esa distribución limpia de perfil de escritorio.
